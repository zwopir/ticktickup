## What this is

TickTickUp is a Wails (v2) desktop app that imports tasks from CSV/JSON files into TickTick via its developer API (TickTick doesn't support CSV import natively). Go backend + Svelte frontend, packaged as a native binary for macOS/Windows/Linux.

## Commands

Prerequisites: Go 1.25+, Node.js 18+, [Wails CLI](https://wails.io/docs/gettingstarted/installation).

```bash
wails dev      # live development (hot-reload frontend + backend)
wails build    # production build -> build/bin/
```

There is no test suite in this repo currently. Frontend-only commands (rarely needed standalone) live in `frontend/package.json`: `npm run dev`, `npm run build`.

CI release builds (`.github/workflows/release.yml`, triggered on `v*` tags) build all four platform targets; Linux requires `-tags webkit2_41` and `libgtk-3-dev libwebkit2gtk-4.1-dev`.

## Architecture

**Backend (Go, package `main`)**
- `main.go` — Wails app entry point, embeds `frontend/dist` and binds `App` methods to the frontend.
- `app.go` — the entire application surface exposed to the frontend (Wails auto-generates JS bindings into `frontend/wailsjs/go/main/App.js` from these exported methods). Holds OAuth flow state, CSV/JSON parsing, and task import orchestration.
- `ticktick/client.go` — TickTick Open API client (OAuth2 code exchange/refresh, `GetProjects`, `CreateProject`, `CreateTask`). No external HTTP dependencies — raw `net/http`.
- `ticktick/storage.go` — reads/writes `~/.ticktickup/config.json` (client id/secret) and `~/.ticktickup/token.json` (OAuth token), both `0600`.

**OAuth flow** (see `app.go`): `StartAuth` spins up a local `http.Server` on `:8765/callback`, opens the system browser to TickTick's consent screen, and blocks in `WaitForAuthCode` on a channel until the callback handler receives the code, then exchanges it for a token and shuts the local server down. Token refresh is attempted transparently in `NeedsAuth`.

**Frontend (Svelte 3 + Vite)**
- Single-component app: `frontend/src/App.svelte` drives a simple state machine via a `screen` variable: `loading` → `config` (enter client id/secret) → `auth` (OAuth consent) → `main` (drag-and-drop import UI, project picker, list creation).
- Calls into Go through the generated bindings in `frontend/wailsjs/go/main/App.js` — these are regenerated by `wails build`/`wails dev`, don't hand-edit them.

**Task input parsing** (`app.go`): both CSV and JSON import paths converge on the shared `TaskInput`/`SubtaskInput` structs before being converted to `ticktick.Task`/`ticktick.CheckItem` for the API call. CSV column matching is case-insensitive with fallback aliases (e.g. `title`/`name`/`task`); subtasks in CSV use a `title|startDate|dueDate` pipe format, semicolon-separated. See README.md for the full field reference and examples — keep it in sync if these formats change.

**Dates**: TickTick's API returns due/start dates as Unix-ms timestamps but accepts ISO strings on input. `ticktick.FlexibleDate` (in `client.go`) has a custom `UnmarshalJSON`/`MarshalJSON` to absorb that asymmetry — reuse it for any new date field rather than adding another one-off type.

## Configuration

Runtime config lives outside the repo in `~/.ticktickup/`: `config.json` (TickTick OAuth client id/secret, user-supplied at first run) and `token.json` (managed automatically). Redirect URI is fixed at `http://localhost:8765/callback` and must match what's registered in the TickTick Developer Portal.
